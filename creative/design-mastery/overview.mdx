## Key principles

<AccordionGroup>
  <Accordion title="Hierarchy" icon="arrow-up-wide-short">
    Visual hierarchy guides the viewer's eye through a composition in a deliberate order. Establish hierarchy through size, weight, color contrast, and spatial positioning. The most important element should be immediately apparent without scanning.
  </Accordion>
  <Accordion title="Consistency" icon="clone">
    Consistent use of type scales, color tokens, spacing units, and component patterns reduces cognitive load and builds user trust. A design system enforces consistency at scale across a product or brand.
  </Accordion>
  <Accordion title="Contrast" icon="circle-half-stroke">
    Contrast creates distinction between elements and ensures accessibility. Apply it to color, size, weight, and shape. WCAG AA requires a minimum contrast ratio of 4.5:1 for normal text.
  </Accordion>
  <Accordion title="Whitespace" icon="expand">
    Whitespace (negative space) is an active design element, not an absence of content. Generous whitespace improves legibility, reduces visual noise, and communicates quality and confidence.
  </Accordion>
  <Accordion title="Alignment" icon="align-left">
    Every element should align to an underlying grid or to another element. Consistent alignment creates visual order and makes compositions feel intentional rather than arbitrary.
  </Accordion>
</AccordionGroup>

## Design system checklist

<Steps>
  <Step title="Define design tokens">
    Establish your foundational tokens: color palette, type scale, spacing scale, border radii, and shadow levels. Store these as variables so they can be updated globally.
  </Step>
  <Step title="Build a component library">
    Create reusable components (buttons, inputs, cards, modals) that consume your tokens. Document usage rules and variants for each component.
  </Step>
  <Step title="Document patterns">
    Record common interaction patterns and layout templates. Include do/don't examples so designers and developers apply patterns correctly.
  </Step>
  <Step title="Establish governance">
    Define a process for proposing, reviewing, and deprecating components. A design system without governance drifts and fragments over time.
  </Step>
  <Step title="Audit regularly">
    Review the system periodically against live product screens. Identify drift, inconsistencies, and gaps, then update documentation accordingly.
  </Step>
</Steps>
